# Problema de investigación

En las Redes de Sensores Inalámbricos (WSN), los nodos de borde capturan información física de manera asíncrona. Si estos datos se transmiten al medio físico en su formato crudo, se generan fallas críticas de enlace como colisiones por acceso simultáneo, desincronización en el receptor y un alto índice de pérdida de paquetes. Debido a que la retransmisión por fallas representa uno de los procesos de mayor consumo energético, el problema fundamental radica en la baja eficiencia y confiabilidad del enlace de comunicación.

Como solución, se propone el empaquetado estructurado de los datos mediante un sistema lógico digital implementado sobre un ESP32-S3. Este sistema encapsula la carga útil proveniente de los sensores junto con cabeceras de sincronización (SHR) y bits de control, conformando una trama estandarizada antes de transmitirla hacia el canal de telecomunicaciones.

---

# Objetivos de la investigación

## Objetivo General

Diseñar e implementar un sistema digital estructurador de tramas de telemetría basado en una arquitectura FSM + Decodificador + Memoria, con el propósito de mejorar la eficiencia y confiabilidad de transmisión en sistemas WSN.

---

## Objetivos Específicos

- Implementar una Máquina de Estados Finitos (FSM) que controle secuencialmente las fases de captura, decodificación, empaquetado y transmisión de datos.

- Implementar un circuito decodificador combinacional que, recibiendo el valor de un bus de direcciones proveniente de un contador, genere señales exclusivas de selección de memoria.

- Evaluar el diseño de hardware mediante la validación experimental de métricas cuantitativas relacionadas con el rendimiento y la temporización.

- Estructurar el subsistema de almacenamiento definiendo una partición ROM estática para almacenar cabeceras y una RAM dinámica que funcione como buffer temporal de telemetría.

---

# Diagrama de Bloques Preliminar

El sistema propuesto corresponde a un ensamblador de tramas de telemetría encargado de capturar datos provenientes de sensores, almacenarlos temporalmente, estructurarlos mediante cabeceras predefinidas y transmitirlos a través de una interfaz UART.

La arquitectura se divide en dos bloques principales:

- Ruta de Datos
- Ruta de Control

---

# Ruta de Datos

El sistema inicia con un sensor que se comunica mediante protocolo I2C utilizando las líneas SDA y SCL. Los datos son recibidos por una interfaz I2C encargada de adaptar la comunicación serial hacia un formato de bus paralelo.

Los datos capturados son almacenados en una memoria RAM, permitiendo guardar múltiples muestras para su posterior procesamiento. Paralelamente, el sistema incorpora una memoria ROM donde se almacenan cabeceras fijas de la trama, como:

- Campos de sincronización
- Tipo de mensaje
- Identificadores
- Bits de control

Para seleccionar entre los datos provenientes de la RAM y las cabeceras almacenadas en ROM se utiliza un multiplexor (MUX). Este bloque es controlado por la FSM y determina qué información será enviada al siguiente módulo.

La salida del multiplexor se dirige a un registro de desplazamiento encargado de convertir datos paralelos en una secuencia serial.

Finalmente, los datos son transmitidos mediante un módulo UART TX, el cual constituye la salida principal del sistema.

---

# Ruta de Control

La ruta de control coordina el funcionamiento integral de todos los bloques del sistema.

El elemento principal es la Máquina de Estados Finitos (FSM), la cual actúa como unidad central de control, recibiendo señales de entrada y generando señales de control para sincronizar las operaciones del sistema.

El direccionamiento de memoria se realiza mediante un contador de direcciones que genera secuencias binarias para recorrer las posiciones de memoria RAM.

Estas direcciones son procesadas por un decodificador lógico, cuya función es activar exclusivamente una posición de memoria en cada instante.

La FSM controla:

- Acceso a memoria
- Selección de datos
- Operación del multiplexor
- Control del registro de desplazamiento
- Transmisión UART

---

# Plataforma Seleccionada

Se seleccionó el microcontrolador **ESP32-S3** debido a las siguientes características:

- Arquitectura de alta velocidad
- Capacidad para ejecutar FSM complejas con temporización estricta
- Amplia memoria SRAM
- Interfaces hardware integradas
- GPIO de alta velocidad
- Compatibilidad con UART e I2C

Estas características permiten implementar tanto la lógica de control como los mecanismos de instrumentación necesarios para obtener métricas temporales en microsegundos.

# Métricas Propuestas

## Latencia de Ensamblado

Se medirá el tiempo transcurrido desde la recepción del último byte de telemetría hasta que la trama esté completamente lista en el registro de salida.

La medición se realizará utilizando:

- Temporizadores internos
- GPIO de monitoreo
- Registros de tiempo en microsegundos

---

## Eficiencia de Trama

Se evaluará la relación porcentual entre:

- Datos útiles (payload)
- Tamaño total de la trama transmitida

La evaluación considerará:

- Cabeceras
- Bits de control
- Datos de sincronización
- Sobrecarga introducida por el empaquetado

---

## Tiempo de Acceso a Memoria

Se medirá el intervalo comprendido entre:

1. Activación de la señal de lectura por parte de la FSM
2. Disponibilidad del dato en la salida de memoria

Esta métrica permitirá verificar:

- Sincronización del subsistema de memoria
- Correcto funcionamiento del decodificador
- Rendimiento temporal del acceso RAM/ROM

---

## Throughput de Transmisión

Se analizará la cantidad de información transmitida exitosamente por unidad de tiempo a través de la interfaz UART.

La medición considerará:

- Número total de bits transmitidos
- Tiempo total requerido para completar la transmisión de una trama
